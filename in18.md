To minimize merge conflicts with upstream:

1. Use Rails i18n system - Keep translations in separate locale files (config/locales/zh-CN.yml)
2. Avoid modifying upstream files where possible - use decorators/concerns for extending models
3. Keep view changes minimal - Use helper methods for translatable strings rather than inline changes
4. Document your changes - Comment where you've made i18n modifications

Actions:

1. Add rails-i18n gem to Gemfile
2. Create i18n initalizer with locale configuration. -> config/initializers/i18n.rb 
3. Create zh-CN locale file witn initial structure. -> config/locale/zh-CN.yml
4. Create LocaleSwitcher concern for controllers. -> app/controllers/concerns/locale_switcher.rb 
	- This allows users to set their preferred locale via:
	- 1. URL parameter (?locale=zh-CN)
	- 2. Session storage (persists across requests)
	- 3. Accept-Language header (browser preference)
	- 4. Falls back to default locale
